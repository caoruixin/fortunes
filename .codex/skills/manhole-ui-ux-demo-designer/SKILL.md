# Manhole UI/UX Demo Designer

Read first:
- `AGENTS.md`
- `references/demo-requirements.md`
- `references/domain-glossary.md`
- existing UI files or `docs/generated/UI_SPEC.md`.

## Design Direction
- Industry feel: municipal roads, engineering, inspection, maintenance.
- Technology feel: AI diagnosis, radar, heatmap, digital traceability.
- Trust feel: metrics, standards, acceptance reports, before/after comparison.
- Demo feel: customer can understand the value in one pass.

## Workflow
1. Define color, typography, layout, map markers, risk scale, cards, tables, timeline, report style.
2. Keep dashboards and workbench screens dense but readable.
3. Make the AI diagnosis and repair simulation visual, not just tabular.
4. Avoid ungrounded decorative visuals and one-note palettes.
5. Verify mobile and desktop layouts do not overlap text or controls.

## Output
Create or update `docs/generated/UI_SPEC.md` with:
- design principles;
- route layouts;
- risk color system;
- component inventory;
- icon and chart guidance;
- demo polish checklist.
